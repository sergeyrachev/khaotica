Flavorc Version 5.x Command Line Reference

Usage

flavorc [options] input.fl

Options

The options which are only available for C++ code generation are in blue, while the ones pertaining to only Java code are in green. The options corresponding to the XML features are in red.

-h or -?
Print a usage message.
 
-V
Print version information.
 
-gc
Generate C++ code (default).
 
-gj
Generate Java code.
 
-gx
Generate XML schema.
 
-n
Parse only; no code will be generated.
 
-g
Do not generate a get() method.
 
-p
Do not generate a put() method.
 
-x
Do not generate a putxml() method.
 
-t
Generate tracing code.
 
-l
Suppress output of line number information for user code (verbatim sections).
 
-oh name
Set the name of the output C++ include file. The default is input.h.
 
-oc name
Set the name of the output C++ file. The default is input.cpp.
 
-oj name
Set the name of the output Java package. The default is input.
 
-og name
Set the name of the Java class that will contain all the global constants. The default is Global.
 
-op name
Set the name of the Flavor run-time library package that will be imported by the generated Java code. The default is flavor.
 
-ox name
Set the name of the output XML schema file. The default is input.xsd.
 
-a size
Set maximum array size. The default is 64, and will be automatically increased whenever the translator can detect it (when the array size is a constant expression).
 
-s
Use null-terminated strings.
 
-msvc
Generate Microsoft Visual C++-style error messages. This enables direct integration with the Microsoft Developer Studio development environment.
 
-i
Don't generate include directives when generating C++ code.
 
-w check
Enable optional checks; use -w help for more information (may be used multiple times).

The following options are available:
            help            print the list of available options
            abstract      warn when pure abstract classes are used
            cyclic         warn when cyclic references are detected
            includes      warn when cyclic includes/imports are detected
 
-B string
Use string as the bitstream I/O class. The default is Bitstream.
 
-F string
Use string as a prefix for internal Flavor-generated variables. The default is _F_.
 
-E string
Use function string to report bitstream syntax errors. The default is flerror.
 
-T string
Use function string to create trace reports. The default is trace.
 
--
This options is used to disallow usage of further options. This allows to use a Flavor source file name that starts with a dash.
